A C I D

Atomicity

Ensures that all operations within a transaction are completed successfully. If any part of the transaction fails, the entire transaction fails and the database state is left unchanged.

Consistency

Guarantees that a transaction can only bring the database from one valid state to another, maintaining database rules such as unique constraints.

Isolation

Ensures that concurrent execution of transactions leaves the database in the same state as if the transactions were executed sequentially.

Durability

Ensures that once a transaction has been committed, it will remain so, even in the event of a power loss, crash, or error.
